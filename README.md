# Отчёт

## Идея решения

Реализовано консольное приложение на C# для учёта финансов. Использована архитектура Simplified VIP. 

### Функционал

* **Создание, редактирование, удаление и просмотр аккаунтов, категорий и операций**
* **Импорт данных из JSON и CSV** (Примеры файлов для импорта лежат в bin/Debug/net8.0)
* **Замер времени выполнения пользовательских сценариев** (кроме импорта)

### Архитектура

* `Domain` - Бизнес-логика
* `Infrastructure` - Доступ к данным, файлам, БД
* `Presentation` - Пользовательский интерфейс, взаимодействие с пользователем
* `Application` - Реализует прикладные сценарии (как бизнес-объекты используются)

## Принципы SOLID и GRASP

### SOLID

* SRP - У каждого класса только одна причина для изменения: BankAccountFacade - только операции с аккаунтами; AccountFactory - только создание аккаунтов и т.д.
* OCP - Добавление новых форматов для импорта не требует изменения старых классов, достаточно унаследоваться от AbstractImporter
* LSP - Реализации ICommand, IConsoleManager и т.д. могут заменять друг друга без нарушения логики (Реализация ICommand заменяет реализацию ICommand и т.д.)
* ISP - Интерфейсы разделены по назначению: ICommand - говорит о том что объект можно использовать как команду, IIdManager - объект может предоставлять уникальные id
* DIP - Классы работают с абстракциями а не с конкретными реализациями: BankAccountFacade использует реализацию IAccountFactory и реализацию IBankAccountRepository

### GRASP

* High Cohesion - Каждый объект делает одно логически цельное дело: BankAccountFacade - работа только с аккаунтами и т.д.
* Low Coupling - Зависимости от абстракций -> выполняется

## Реализованные паттерны

* Фабрика - BankAccountFactory, OperationFactory, CategoryFactory - Позволяют создавать доменные объекты
* Фасад - BankAccountFacade, OperationFacade, CategoryFacade - Позволяют работать с данными об аккаунтах, операциях и категориях используя простой интерфейс
* Команда - Позволяет представлять действия как объект. Позволяет удобно использовать декораторы. Интерфейс ICommand реализуют на каждом экране по 4 команды - Просмотр, добавление, редактирование, удаление
* Декоратор - Не изменяет реализацию комманды, но позволяет добавить новый функционал (Конкретно в данном случае замер времени исполнения команды). BaseDecorator и TimerDecorator
* Шаблонный метод - Позволяет изменять только часть метода так как не всегда нужно менять его полностью. AbstractImporter реализует паттерн, те кто наследуется изменяют только те методы что нужны им для работы
* Строитель (Builder) - У каждого экрана есть свой билдер который скрывает сложную логику создания. Также используется StringBuilder который сам по себе реализует данный паттерн.
* Стратегия (Частично) - Выбор конкретного импорта по формату в интеракторе
